# Two-Person Conventions

Written generically. Never hard-code personal names — resolve from
the operating space's `members` list in config.

- **Shared tasks always get an Assignee.** A task in a shared space with no
  assignee is incomplete; ask or infer, don't leave it blank.
- **Waiting = blocked on your partner.** Use the Waiting status (In progress group)
  when a task can't proceed until the other person acts.
- **Capture routes to a shared space only when that space is named or clearly
  implied.** Otherwise capture is personal. When ambiguous, default personal.
- **Never attempt writes into a partner's private area.** They're invisible to you
  anyway (Notion enforces per-user permissions) — don't try and don't fake it.
- **Resolve "assign to <partner>"** from the operating shared space's member list;
  if the name isn't a member, say so rather than guessing an ID.
